1. 不支持text类型，使用CLOB
2. 为表添加注释，使用单独语句添加：COMMENT ON TABLE "TEST"."TBL_TEST" IS '测试用表';
3. 特殊表名、列名的声明，使用双引号：CREATE TABLE "desc" ( "desc" varchar(256) );
4. varchar 类型长度含义不同
	- 达梦中 varchar 的长度计算的是字节数 
	- 所以，达梦中 varchar(128) 是存不下 128 个汉字的，最多只能存 42 个（跟字符集有关） 
	- 达梦中的 length 函数计算的也是字符数，如果要计算字节数，应该使用函数：OCTET_LENGTH
	
	
在达梦数据库中，表空间（Tablespace）和模式（Schema）是两个不同的概念。
	表空间（Tablespace）：表空间是用于管理和组织数据库中的数据的逻辑结构。它是数据库中存储表、索引和其他对象的物理空间。在达梦数据库中，表空间可以包含多个数据文件，这些数据文件实际上是用来存储表和索引数据的文件。
	模式（Schema）：模式是在数据库中用于组织和管理对象的逻辑结构。它是一种命名空间，用于在数据库中分隔和组织不同用户或应用程序的对象。在达梦数据库中，模式是一种逻辑容器，可以包含表、视图、存储过程、函数等数据库对象。
	表空间和模式之间的关系是，表空间用于管理物理存储，而模式用于管理逻辑结构。一个数据库可以包含多个表空间，每个表空间可以包含多个模式，而每个模式可以包含多个表和其他数据库对象。
	通过使用表空间和模式，可以更好地组织和管理数据库中的数据和对象。表空间提供了物理存储的管理，而模式提供了逻辑分隔和命名空间的管理，使得不同用户或应用程序之间的对象可以独立存在并互不干扰
